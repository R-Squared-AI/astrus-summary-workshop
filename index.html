Six distinct, production-realistic design concepts for the at-a-glance submission summary — the first panel an underwriter sees at the top of an Astrus submission in Salesforce Lightning. This gallery presents candidate designs for a single high-signal panel that answers an underwriter's first question in seconds: what came in, what's solid, and what's missing — and, when a gap exists, how to trigger a targeted, underwriter-controlled reprocess without leaving the summary. The running example is Josh's exact anti-scenario: a submission that looks healthy but is missing the loss run he needs, followed by a new loss-run file landing on the record. Every concept tells that story: flag the gap → a new file arrives → pick per-LOB scope → confirm → reprocess (a populated Bound Premium stays locked).

Standalone panel / tab pole

Concepts 01, 02, 04, 05 — a dedicated summary component that sits above the record. Best if the summary should be its own place.

Field-evidence extension pole

Concepts 03 & 06 — extend the existing data-quality / field-evidence UI so the summary lives inside the trust surface underwriters already use.

How to read these: Open any concept below — each is a self-contained, SLDS-styled mockup wrapped in a faux Lightning record page, all rendering the same SUB 022667 data so you can compare designs head-to-head. Look for (1) how fast the headline reads, (2) how present/partial/missing is signalled, and (3) where the reprocess control lives. The tab-vs-panel spread above is deliberate — Josh & Rich left that decision open, so we span both poles for you to choose.

The six concepts

These are static HTML/CSS/JS mockups of a future Lightning Web Component — hand-rolled to match SLDS tokens, type ladder, and confidence bands, with no external frameworks or CDNs. Every pattern shown maps 1:1 to a real LWC (base Lightning components, Lightning Data Service wires, inline SVG/CSS charts) so the winning concept is realistic to build.
