Concept 06 — "Coverage Map"

Thesis: A spatial overview. A state × line-of-business matrix shows what Astrus produced and where — so the "Hawaii needs its own policy" shape is legible at a glance — with the headline exposures and the loss-run flag arranged quietly around it.

What it does NOT do: it never invents a completeness or quality verdict. Vehicle counts are shown as numbers (31 CA · 2 HI), not a "fleet looks incomplete" judgment. A grey cell means the line simply is not on the submission for that state — not a problem.

Target underwriter job
"Show me — boom, boom, boom — which lines came in, with the exposure figure that proves it, and flag the loss run I don't have. Then show me the CA-vs-HI picture."
Positioning
Standalone summary panel pinned to the top of the submission record — the first thing the underwriter reads.
Why it's honest
Every cell maps to a real field: green = exposure populated; amber = a named field is empty (no effective date); red = requested but nothing produced (loss runs); grey = not on submission. No confidence, no fabricated completeness.
LWC feasibility
Component c-astrus-coverage-map: lightning-card + a semantic CSS-grid matrix (for:each rows × state columns), status via lightning-icon, money via lightning-formatted-number. Data from LDS getRecord + one imperative Apex wire getCoverageByState. Inline SVG/CSS, LWS-safe — no CDN, no fonts, no external libs.
